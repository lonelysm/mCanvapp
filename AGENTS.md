## 프로젝트 개요
- **목적**: JSON·CSV 등 외부 데이터를 불러와 캔버스에서 테이블·UML·Blueprint 스타일 노드 등 다양한 형태로 시각화하고, 캔버스 내용을 다시 데이터로 익스포트하는 도구.
- **스택**: 백엔드 Python Flask, 프론트 Jinja2 템플릿, 캔버스·UI 로직은 바닐라 JS(ES 모듈). 소규모·낮은 진입장벽 유지, 대형 프레임워크 없음. 보안·인증은 이 프로젝트 범위가 아님.

---

## 아키텍처 원칙

- **백엔드**: Flask가 CSV/JSON 읽기·변환·API 제공. 데이터 소스 → 중간 구조(리스트/딕셔너리 등) 변환은 서버에서 수행.
- **프론트**: Jinja2로 초기 HTML·초기 데이터 주입. 캔버스·드래그·뷰 전환·익스포트는 JS에서 처리.
- **데이터 흐름**:
  - 임포트: 파일/업로드 → Flask 파싱 → (JSON 응답 또는 Jinja2 변수) → JS가 테이블/UML/노드 뷰로 렌더링.  
  - 익스포트: JS가 캔버스 상태(노드, 테이블, 연결)를 구조화된 객체로 직렬화 → Flask로 전달해 JSON/CSV 등으로 저장.

---

## 코딩 규칙

- **주석**: 한글 사용. setter getter를 제외한 함수 및 멤버함수에는 구체적으로 어떤 일을 하는지 주석 남김
- **로그**: 모든 I/O 이벤트에는 시작 및 종료시에는 반드시 로그를 남기고 실패시 로그도 남김
- **null/undefined/None 처리**: null일 때는 로그를 남기고 early return 등으로 처리. 가능하면 if로 한 번 더 감싸지 말고 명시적으로 처리.
- **중복**: 최소화. 공통 로직은 함수/모듈로 분리.
- **메모리·성능**: 불필요한 객체 생성·반복 연산을 줄일 것.
- **들여쓰기**: 4칸 스페이스.
- **변수명**: `s`, `i` 같은 한 글자보다 역할이 드러나는 이름 사용.

## 하위 프로젝트별 규칙

이 워크스페이스는 여러 하위 폴더로 구성될 수 있습니다. **특정 하위 폴더 내 파일을 수정·추가할 때는 반드시 해당 폴더의 `AGENTS.md`를 먼저 읽고, 그 안의 프로젝트별 규칙·결정 사항을 적용한 뒤 작업한다.**
